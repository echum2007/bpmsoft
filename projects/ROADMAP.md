## Волна 2 — Уведомления по почте

Одна тема — удобно делать серией без переключения контекста.

**Архитектурное решение (15.04.2026):** все задачи 2.1–2.5 реализуются через единый механизм `UsrEmployeeNotificationManager` + справочник `UsrEmployeeNotificationRule`. Задача 2.0 — обязательный первый шаг. Подробности: `notifications-wave2/ANALYSIS.md`, раздел 5.

| # | Задача | Сложность | Приоритет | Примечание |
| --- | ------ | --------- | --------- | ---------- |
| **2.0** | **UsrEmployeeNotificationManager** — ядро: C#-класс + справочник `UsrEmployeeNotificationRule` | Средняя | **Первый шаг** | Фундамент для 2.1–2.5. Аналог `SendEmailToCaseStatusChanged` для сотрудников |
| 2.3 | Email сотруднику при ответе клиента — текст письма + срабатывание из любого статуса | Средняя | Высокий | Первое событие в новом механизме. Закрывает GAP с фильтром по статусам («Новое»/«В работе» сейчас не уходит) |
| 2.2 | Зависание в «Получен ответ» — напоминание + эскалация | Средняя | Высокий | Фоновый процесс, событие=Зависание |
| 2.4 | Email сотруднику о смене статуса (эскалация, переоткрытие, отмена и др.) | Средняя | Средний | Событие=СменаСтатуса, EventListener на Case.OnUpdated |
| 2.1 | SLA-пороги 75/85/95% — предупреждение до просрочки | Средняя–Высокая | Средний | Фоновый процесс + `UsrSlaNotificationLog` для дедупликации |
| 2.5 | Роль наблюдателя — подписка без назначения ответственным | Высокая | Средний | Новый объект `UsrCaseWatcher`, деталь на странице, рассылка через менеджер |
| 2.6 | Текст ответа инженера в уведомлении **клиенту** | Низкая | Низкий | Invokable-макрос `[#@Invoke.UsrLastEngineerReply#]` в шаблоне «Сообщение о получении нового ответа» |
| 2.7 | Имя инженера в подписи письма клиенту | Низкая | Низкий | Макрос `[#Owner.Name#]` в подвал шаблонов, без кода |
