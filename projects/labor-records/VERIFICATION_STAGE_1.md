## Находки (нужны правки или решения)

### 🔴 Находка 1.1 — Раздел 0 (Резюме) ссылается на ICalendarRepository

**Проблема:** в пункте 6 резюме:
> Определение "формально рабочего времени" — платформенное API `ICalendarRepository` + `ICalendar.IsWorkTime(DateTime)` (namespace `BPMSoft.Configuration.Calendars`). Каскад календарей: `ServicePact → ServiceItem → BaseCalendar`.

Это **устаревшая информация**. В разделе 3.4 уже исправлено на `TermCalculatorActions` (2026-04-25, R3 снят). Резюме забыли обновить.

**Правка:** заменить в разделе 0:
- `ICalendarRepository` → `TermCalculatorActions` (пакет SLM)
- `IsWorkTime` → `IsTimeInWorkingInterval`
- `namespace BPMSoft.Configuration.Calendars` → `namespace BPMSoft.Configuration`

### 🟡 Находка 1.2 — Серверная валидация длительности не добавлена в T-07

**Проблема:** замечание 2.2 #4 требует правило "длительность ≥ 1 минута". В T-10 (popup JS) валидация есть. В T-12 упомянуто: "Лучше реализовать через валидатор на EntitySchema, а не через бизнес-правило" и предложен код для `OnSaving`. Но в T-07 (EventListener C#) этот блок НЕ добавлен.

**Последствие:** если пользователь обойдёт popup (через SQL/импорт/деталь с багом) и создаст запись с 0 минут — сервер не остановит. Риск низкий, но для полноты защиты — стоит добавить.

**Правка:** добавить в T-07 (код EventListener) перед Шагом 2 блок проверки длительности:
```csharp
var hours = entity.GetTypedColumnValue<int>("UsrWorkDurationHours");
var minutes = entity.GetTypedColumnValue<int>("UsrWorkDurationMinutes");
if (hours * 60 + minutes < 1) {
    throw new Exception("Длительность работ должна быть не менее 1 минуты.");
}
if (hours < 0 || hours > 23 || minutes < 0 || minutes > 59) {
    throw new Exception("Часы должны быть 0-23, минуты 0-59.");
}
```
