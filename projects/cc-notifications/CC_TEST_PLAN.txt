План тестирования CC-функциональности
=======================================

A. Исходящие письма — подстановка CC
-------------------------------------
A1. CC заполнен только в обращении
     → Письмо уходит с этим адресом в CC

A2. CC заполнен только в сервисном договоре
     → Письмо уходит с адресом из договора в CC

A3. CC заполнен и в обращении, и в договоре — разные адреса
     → Оба адреса в CC, без дублей

A4. CC заполнен и в обращении, и в договоре — одинаковый адрес
     → Адрес в CC ровно один раз

A5. CC пуст и в обращении, и в договоре
     → Письмо уходит без CC (как раньше)

A6. Несколько адресов в CC обращения (a@x.ru; b@x.ru)
     → Оба адреса в CC письма

A7. Повторная отправка по тому же обращению
     → Адреса не дублируются при каждой отправке


B. Входящие письма — сохранение CC в обращение
-----------------------------------------------
B1. Входящее письмо с CC-адресом, в обращении CC не заполнен
     → Адрес сохраняется в поле CC обращения

B2. Входящее письмо с CC, в обращении уже есть тот же адрес
     → Дубль не добавляется

B3. Входящее письмо с CC, в обращении уже есть другой адрес
     → Оба адреса сохраняются в обращении

B4. Входящее письмо с несколькими адресами в CC
     → Все адреса сохраняются в обращении

B5. Входящее письмо без CC
     → Поле CC обращения не меняется

B6. Входящее с именем отправителя: "Иванов Иван <ivanov@x.ru>"
     → Сохраняется только адрес ivanov@x.ru, без имени и скобок


C. Цепочка входящее → исходящее
---------------------------------
C1. Клиент прислал письмо с CC boss@x.ru → оператор отвечает
     → В ответе автоматически стоит boss@x.ru в CC

C2. Клиент прислал с CC boss@x.ru, в договоре admin@x.ru → ответ
     → В CC оба адреса: boss@x.ru и admin@x.ru

C3. Несколько входящих с разными CC → ответ
     → Все уникальные CC-адреса из всех входящих и из договора


D. Валидация ввода — поле CC в обращении и в договоре
------------------------------------------------------
D1. Ввести корректный email: user@example.com
     → Сохранение проходит

D2. Ввести несколько адресов через ";": a@x.ru; b@x.ru
     → Сохранение проходит

D3. Ввести несколько адресов через ",": a@x.ru, b@x.ru
     → Сохранение проходит

D4. Ввести невалидный адрес: not-an-email
     → Ошибка, сохранение заблокировано

D5. Ввести смесь корректных и некорректных: a@x.ru; bad-email; b@x.ru
     → Ошибка с указанием некорректного адреса

D6. Оставить поле CC пустым
     → Сохранение проходит (CC необязательно)

D7. Повторить D1–D6 в карточке договора
     → Аналогичное поведение


E. Граничные случаи
--------------------
E1. Обращение без договора, CC в обращении есть
     → CC подставляется, ошибок нет

E2. Письмо не связано ни с каким обращением
     → Письмо уходит как обычно, CC не затрагивается

E3. Адреса различаются только регистром: User@X.RU в обращении, user@x.ru в договоре
     → В CC один адрес (без учёта регистра)
