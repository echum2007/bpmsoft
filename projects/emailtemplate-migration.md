# EmailTemplate — перенос шаблонов с прода на dev

## Задача

Перенести три отредактированных шаблона EmailTemplate с прода на dev через пакет CTI.

## Три целевых шаблона

| Шаблон | EmailTemplate Id | PreviewImage (SysImage) Id |
|--------|-----------------|---------------------------|
| Сообщение о взятии обращения в работу | `d790ccfe-db7b-4881-913e-b02aab5b8593` | `a7a54bc5-eb66-4818-a0a6-5d83f6acece8` |
| Сообщение о закрытии обращения | `80cdb129-de99-433f-8783-9d71c205607b` | `fb6913c8-bca8-4e9f-a4f1-b8fac970198b` |
| Отправка уведомления ответственному о смене статуса на Получен ответ | `18834f34-0ebc-4392-a313-c21f40ea0d26` | `ee4d4859-97f1-415e-88b7-70afb20aa612` |

## Проблема

FK-ошибка при импорте CTI-пакета:

```
insert or update on table "EmailTemplate" violates foreign key constraint "FKJKkd7aSxQdiN6pbsi1lGVzF9qw"
```

`EmailTemplate.PreviewImageId → SysImage.Id` — записей SysImage с картинками-превью нет на dev.

**Причина:** картинки появились на проде ещё в v1.6 и остались после апгрейда до v1.9. Dev создавался из чистого дистрибутива v1.9, где этих записей нет.

## Решение

Сделать новый экспорт CTI с **двумя секциями**:

1. `EmailTemplate` — только 3 записи (по Id выше)
2. `SysImage` — только 3 записи (по PreviewImage Id выше)

`SysImage` не имеет исходящих FK → дополнительных зависимостей нет, импорт пройдёт чисто.

## Следующий шаг

Экспортировать через утилиту с фильтром по Id для обеих таблиц, положить архив в `src/`.
