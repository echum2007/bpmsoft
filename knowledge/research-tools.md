# Инструменты исследования документации BPMSoft

**Обновлено:** 2026-04-19  
**Методология:** Двухуровневый поиск — документация производителя + собственная база знаний

---

## Методология: два источника для каждой задачи

Для **любого вопроса** используй **оба источника параллельно:**

1. **Документация производителя (Уровни 1-3)** — авторитетный источник официальной информации
2. **База знаний проекта (Уровень 4)** — опыт, нюансы, решения, найденные противоречия

**Уровни 1-3 — это альтернативные пути доступа к одной документации**, выбирай наиболее доступный:
- **Уровень 1** (NotebookLM) — самый эффективный (5-7x экономия токенов)
- **Уровень 2** (edu.bpmsoft.ru) — fallback если NotebookLM недоступен
- **Уровень 3** (локальный PDF) — fallback если сеть недоступна

**Уровень 4 — собственная база знаний** (не альтернатива, а дополнение):
- Смотри **всегда**, независимо от наличия ответа в документации
- Содержит: опыт, найденные баги, нюансы, решения из интернета
- Если найдёшь противоречие с документацией → сообщи, обсудим как исправить

## Уровни доступа к документации производителя

### 🔴 Уровень 1: NotebookLM (ПРИОРИТЕТ для документации)

**Инструмент:** `mcp__notebooklm__ask_question`  
**ID блокнота:** `bpmsoft-documentation`  
**URL:** https://notebooklm.google.com/notebook/eb410184-caa8-42fe-92b4-6f1971f4425f

**Характеристики:**
- ✅ Полная документация BPMSoft 1.9
- ✅ Ответы с цитатами и источниками (от Gemini)
- ✅ Минимум токенов: ~827 на запрос
- ✅ Быстрый поиск и навигация
- ✅ Сессийная память (до 10 вопросов в контексте)

**Эффективность:** 5-7x экономия токенов vs остальные источники

**Когда использовать:** Первый выбор для поиска информации в документации производителя

**Как работает:** MCP открывает браузер, использует твою Google-аутентификацию, задаёт вопрос Gemini, возвращает ответ с цитатами

---

### 🟡 Уровень 2: edu.bpmsoft.ru (FALLBACK #1 для документации)

**Инструмент:** `/bpmsoft-kb` скилл  
**URL:** https://edu.bpmsoft.ru/baza-znaniy/  
**Авторизация:** e.chumak@cti.ru

**Характеристики:**
- ✅ Официальная онлайн-база знаний BPMSoft
- ✅ Всегда актуальная информация
- ⚠️ Браузерная автоматизация (Playwright, хрупко к UI-изменениям)
- ⚠️ Дороже по токенам: ~2000-4000 на запрос

**Когда использовать:** 
- Если NotebookLM недоступен
- Если нужна информация, которой нет в NotebookLM блокноте
- Для уточнения деталей после ответа NotebookLM

---

### 🟠 Уровень 3: Локальный PDF (FALLBACK #2 для документации)

**Инструмент:** Read tool на `Documentation 1.9/`  
**Содержание:** 8 PDF с категоризацией (для разработчика, аналитика, администратора, пользователя)

**Характеристики:**
- ⚠️ Самый дорогой по токенам: ~4000-6000 на запрос
- ⚠️ Требует ручного подбора нужного PDF
- ⚠️ Нет структурированного индекса поиска
- ✅ Работает без сети (файлы локальные)

**Индекс PDF:** [DOCUMENTATION_INDEX.md](DOCUMENTATION_INDEX.md)

**Когда использовать:** Если сеть полностью недоступна и нужна информация из документации производителя

## Собственная база знаний (Уровень 4)

### 🔵 Локальная вики + система памяти

**Пути:**
- `knowledge/wiki/` — структурированная вики по компонентам (notifications, platform, processes и т.д.)
- `knowledge/archive/` — архивные материалы
- `.claude/projects/.../memory/` — автоматическая система памяти

**Содержание:**
- ✅ Найденные нюансы и баги
- ✅ Решения задач с контекстом
- ✅ Информация из интернета и других источников
- ✅ Противоречия между официальной документацией и практикой

**Характеристики:**
- ✅ Экономит токены на известные задачи
- ✅ Содержит опыт и практические решения
- ⚠️ Может быть устаревшей или неполной

**Как использовать:**
- Смотри **всегда**, независимо от наличия ответа в документации
- Дополняй информацию из Уровней 1-3
- Если найдёшь противоречие с официальной документацией → сообщи, нужно будет разбираться

## Практический алгоритм для любой задачи

```
1. Вопрос к задаче → три источника параллельно

2а. ДОКУМЕНТАЦИЯ (выбери первый доступный):
   ├─ NotebookLM (Уровень 1) → быстро + цитаты
   ├─ edu.bpmsoft.ru (Уровень 2) → если Уровень 1 недоступен
   └─ Локальный PDF (Уровень 3) → если нет сети

2б. БАЗА ЗНАНИЙ (смотри всегда):
   ├─ knowledge/INDEX.md → найди релевантные страницы
   ├─ knowledge/wiki/* → прочитай по теме
   └─ .claude/memory/ → проверь автоматическую память

3. Комбинируй ответы:
   ├─ Основа от документации (Уровни 1-3)
   ├─ Дополнение от базы знаний (Уровень 4)
   └─ Если противоречие → сообщи Евгению, нужно обсуждение
```

## Примеры применения

### Пример 1: Стандартный вопрос о документации

**Задача:** Как настроить Google Mail OAuth в BPMSoft?

**Процесс:**
1. NotebookLM: `ask_question` → "Как настроить Google Mail OAuth 2.0?" → 827 токенов → ответ с цитатами
2. Параллельно база знаний: `knowledge/wiki/platform.md` → email-архитектура + `memory/reference_*` → контекст
3. Результат: официальный ответ (OAuth только для Microsoft 365, для Gmail — Basic auth) + практический контекст из вики

### Пример 2: Противоречие между источниками

**Найди в документации:** Feature X работает так-то  
**Найди в вики:** На практике работает иначе (баг или нюанс)

**Действие:** Сообщи Евгению о противоречии, будем разбираться и обновлять базу знаний

## Техническое обслуживание

### Обновление NotebookLM блокнота

Если документация BPMSoft обновилась:
1. Открыть https://notebooklm.google.com
2. Загрузить новые PDF в блокнот `bpmsoft-documentation`
3. Перезагрузить MCP:
   ```bash
   mcp__notebooklm__cleanup_data(confirm=true, preserve_library=true)
   mcp__notebooklm__re_auth(show_browser=true)
   ```

### Проверка здоровья NotebookLM

```
mcp__notebooklm__get_health
```

Ожидается:
```json
{
  "status": "ok",
  "authenticated": true,
  "active_sessions": 0
}
```
