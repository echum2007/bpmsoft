## Мастер разделов

**Открыть новый раздел:** Дизайнер системы → «Мастер разделов»  
**Открыть существующий:** раздел → **«Настройка вида»** → «Открыть мастер раздела»

> ⚠️ Кнопка называется **«Настройка вида»** (не «Вид»)  
> ⚠️ В мастере кнопка **«Сохранить»** (не «Опубликовать»)

**Важно:** Код раздела не меняется после первого сохранения — задавать с префиксом сразу (`Usr`, `CTI` и т.д.).

Автоматически создаются 7 объектов: `<Code>`, `<Code>File`, `<Code>Folder`, `<Code>InFolder`, `<Code>Tag`, `<Code>InTag`, `<Code>RequestsVisa`.

После создания нового раздела — **обязательный перелогин** для отображения в навигации.

## Структура страницы

```
Страница → Вкладки → Блоки → Поля
```

- **ProfileContainer** — блок в «шапке» страницы. ⚠️ Не поддерживает два поля в одну строку
- **ColumnContainer** — многоколоночный блок на вкладках, гибкая раскладка

Нельзя удалить все вкладки — минимум одна. Вкладку «Лента» при удалении можно восстановить только через код.

## Бизнес-правила

**Путь:** Мастер раздела → «Бизнес-логика» → «Бизнес-правила»

### Доступные действия

| Действие | Описание |
|---|---|
| Показывать элемент | Показать/скрыть поле или вкладку |
| Делать поле редактируемым | Включить/заблокировать редактирование |
| Делать поле обязательным | Обязательное для заполнения |
| Добавлять фильтр значений | Фильтровать lookup-поле по условию |
| Заполнять значение поля | Автоматически подставить значение |

> ⚠️ Условия с типом «Дата» и «Дата и время» не поддерживаются в правилах цветового выделения.

## Системные настройки (SysSettings)

**Путь:** Дизайнер системы → «Системные настройки»

```javascript
// JavaScript — чтение
BPMSoft.SysSettings.querySysSettingsItem("UsrSLAResponseTime", function(value) {
    console.log(value);
}, this);
```

```csharp
// C# — чтение
var hours = BPMSoft.Core.Configuration.SysSettings.GetValue<int>(
    UserConnection, "UsrSLAResponseTime");

// C# — строка с default
var email = BPMSoft.Core.Configuration.SysSettings.GetValue<string>(
    UserConnection, "SupportServiceEmail", string.Empty);
```

## JavaScript AMD-модуль (ClientUnitSchema)

Замещение страницы — тип схемы «Клиентский модуль», ExtendParent.

```javascript
define("CasePage", ["BPMSoft", "ext-base"],
function(BPMSoft, Ext) {
    return {
        entitySchemaName: "Case",
        diff: [
            {
                "operation": "insert",
                "parentName": "Header",
                "propertyName": "items",
                "name": "UsrCcEmails",
                "values": {
                    "layout": { "column": 0, "row": 5, "colSpan": 12 },
                    "bindTo": "UsrCcEmails"
                }
            }
        ],
        methods: {
            myMethod: function() {
                var value = this.get("FieldName");
                this.set("FieldName", newValue);
            }
        }
    };
});
```

## Нюансы и известные ловушки

### ProfileContainer — ограничение LABEL

`ProfileContainer` использует специфический `ViewGenerator` и **не поддерживает** `itemType: 6` (LABEL) — элемент молча игнорируется.

**Решение:** использовать `Container` (`itemType: 7`) и задавать текст через `innerHTML` в методе `render()` модуля страницы.

---

### Ловушки binding (visible)

- **Строковые атрибуты** не работают как boolean: `visible: { bindTo: "UsrStatusName" }` не сработает, даже если строка «Active» — парсер ожидает явный boolean.
- **Вычисляемые методы** могут не пересчитываться, если зависимости не объявлены в модели явно. Безопаснее использовать вспомогательный boolean-атрибут с явным `changed`-хендлером.

---

### Диагностика загрузки CSS (AMD)

Префикс `css!` в массиве зависимостей `define` обязателен. Без него stylesheet молча игнорируется.

**Диагностика:** открыть DevTools → Console, найти строку `CSS rules found:` для целевого файла. Пустой массив `[]` — признак отсутствия префикса `css!`.
