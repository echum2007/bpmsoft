# No-code кастомизация BPMSoft

## Мастер разделов

**Как открыть:**
- Новый раздел: Дизайнер системы → «Мастер разделов»
- Существующий раздел: открыть раздел → «Настройка вида» → «Открыть мастер раздела»

**Важно:** в мастере кнопка называется «Сохранить» (не «Опубликовать») — изменения применяются автоматически.

## Создание раздела

**Код раздела не меняется после первого сохранения** — задавать с префиксом сразу (`CTI`, `Usr` и т.д.).

Автоматически создаются 7 объектов: `<Code>`, `<Code>File`, `<Code>Folder`, `<Code>InFolder`, `<Code>Tag`, `<Code>InTag`, `<Code>RequestsVisa`.

После создания — **обязательный перелогин** для отображения в навигации.

## Структура страницы

```
Страница → Вкладки → Блоки → Поля
```

- **ProfileContainer** — блок в «шапке» страницы. ⚠️ Не поддерживает два поля в одну строку
- **ColumnContainer** — многоколоночный блок на вкладках, гибкая раскладка

Нельзя удалить все вкладки — минимум одна. Вкладку «Лента» при удалении можно восстановить только через код.

## Бизнес-правила

**Путь:** Мастер раздела → «Бизнес-логика» → «Бизнес-правила»

### Доступные действия

| Действие | Описание |
|---|---|
| Показывать элемент | Показать/скрыть поле или вкладку |
| Делать поле редактируемым | Включить/заблокировать редактирование |
| Делать поле обязательным | Сделать обязательным для заполнения |
| Добавлять фильтр значений | Фильтровать список lookup-поля по условию |
| Заполнять значение поля | Автоматически подставить значение |

### Примеры для Service Desk

```
УСЛОВИЕ: «Приоритет» = «Критический»
ДЕЙСТВИЕ: поле «Ответственный» → сделать обязательным

УСЛОВИЕ: «Тип» = «Инцидент»
ДЕЙСТВИЕ: вкладка «SLA» → показывать

УСЛОВИЕ: «Сервисный контракт» заполнен
ДЕЙСТВИЕ: поле «Категория» → фильтровать значения по контракту
```

⚠️ Условия с типом «Дата» и «Дата и время» не поддерживаются в правилах цветового выделения.

## Правила цветового выделения

**Путь:** Рабочее место «Конструктор» → «Правила цветового выделения»

Ключевые параметры: Объект, Область (строки в разделе / поля на странице), Условие, Цвет, Роль пользователя, Приоритетность.

Примеры для Service Desk:
- Обращения с критическим приоритетом → красный фон
- Нарушение SLA → оранжевый фон
- Закрытые обращения → серый шрифт

## Системные настройки (SysSettings)

**Путь:** Дизайнер системы → «Системные настройки»

- `SysSettings` — метаданные (Код, Название, Тип)
- `SysSettingsValue` — значения

```javascript
// JavaScript
BPMSoft.SysSettings.querySysSettingsItem("UsrSLAResponseTime", function(value) {
    console.log(value);
}, this);
```

```csharp
// C#
var hours = BPMSoft.Core.Configuration.SysSettings.GetValue<int>(
    UserConnection, "UsrSLAResponseTime");
```

## Рабочие места

**Путь:** Дизайнер системы → «Настройка рабочих мест»

Типичные рабочие места Service Desk:
- **Оператор:** Обращения, Контакты, Очередь
- **Менеджер:** Обращения, Итоги, Сервисные контракты, SLA
- **Администратор:** всё + Конструктор

## Продвинутый дизайнер интерфейсов (Angular)

**Путь:** Дизайнер системы → «Продвинутый дизайнер интерфейсов»

Основан на Angular (в отличие от классического мастера на ExtJS). Поддерживает кастомные Angular-компоненты. Направление развития UI в BPMSoft.
