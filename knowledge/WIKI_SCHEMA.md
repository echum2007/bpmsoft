## Структура вики

```
knowledge/
├── WIKI_SCHEMA.md          ← этот файл: правила, операции, лимиты
├── INDEX.md                ← навигационный индекс всех страниц
│
├── wiki/                   ← основные страницы знаний
│   ├── platform.md         ← платформа BPMSoft: стек, архитектура, пакеты
│   ├── cti-package.md      ← пакет CTI: схемы, UId, архитектурные решения
│   ├── notifications.md    ← система уведомлений: процессы, email, архитектура
│   ├── processes.md        ← BPMN, Script Task, EventListener, DCM
│   ├── ui-customization.md ← страницы, детали, мастер разделов, no-code
│   ├── data-model.md       ← объекты, колонки, ESQ, ключевые таблицы БД
│   ├── troubleshooting.md  ← известные проблемы, диагностика, кейсы
│   └── projects/           ← страницы по активным проектам
│       ├── cc-notifications.md
│       ├── service-mode-indicator.md
│       ├── notifications-wave2.md
│       └── labor-records.md
│
└── archive/                ← устаревшие файлы (Actuel/Archive паттерн)
    └── YYYY-MM-DD_<name>.md
```

## Операции

### Ingest (конец каждой сессии)

Когда выяснено что-то новое о платформе или проекте:

1. Определить релевантную страницу вики (`wiki/*.md` или `wiki/projects/*.md`)
2. Обновить страницу: добавить, уточнить или исправить информацию
3. Запустить **Lint** (см. ниже)
4. Обновить `INDEX.md` если добавлена новая страница
5. Обновить поле `Обновлено:` в заголовке изменённой страницы

### Query (поиск по вики)

При получении задачи читать в следующем порядке:
1. `INDEX.md` — найти релевантные страницы
2. `wiki/cti-package.md` — актуальные UId и схемы
3. Релевантные тематические страницы
4. При необходимости — обратиться к источникам

### Lint (автоматически после Ingest)

Проверить:
- [ ] UId в `wiki/cti-package.md` консистентны между собой
- [ ] Нет страниц старше **60 дней** без обновления → пометить `⚠️ требует проверки`
- [ ] Нет оборванных ссылок между страницами вики
- [ ] Статусы проектов в `wiki/projects/` актуальны
- [ ] `INDEX.md` содержит все страницы из `wiki/`

## Правила ведения страниц

### Формат заголовка каждой страницы

```markdown
# [Название страницы]

**Обновлено:** YYYY-MM-DD  
**Источники:** [список источников]

---
```

### Лимиты

| Параметр | Лимит |
|---|---|
| Размер одной страницы | ≤ 500 строк |
| Размер страницы проекта | ≤ 200 строк |
| Глубина вложенности разделов | ≤ 3 уровня (`###`) |

Если страница превышает лимит — разбить на подстраницы или перенести детальный код в `projects/`.

### Что хранить в вики

✅ **Хранить:**
- Синтезированные знания о платформе (паттерны, поведение API)
- UId схем и метаданные CTI (но не сам код)
- Архитектурные решения с обоснованием
- Известные подводные камни и проблемы
- Статусы и ключевые факты по проектам

❌ **Не хранить:**
- Исходный код CTI (он в архиве `src/`)
- Копии PDF-документации
- Временные заметки текущей сессии
- Что уже есть в CLAUDE.md

### Actuel/Archive паттерн

При устаревании информации:
- Не удалять — переместить в `archive/YYYY-MM-DD_<name>.md`
- В основном файле оставить ссылку: `> Устарело: см. archive/YYYY-MM-DD_<name>.md`
