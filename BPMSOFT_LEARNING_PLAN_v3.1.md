# План обучения разработке и кастомизации BPMSoft
## Версия 3.1 — с философией обучения и привязкой ко всем доступным документам

**Для кого:** специалист без опыта программирования  
**Цель:** освоить технологический стек BPMSoft (JavaScript, C#, SQL, ExtJS, HTML/CSS, LESS, AMD-модули, OData, REST, BPMN) и научиться кастомизировать платформу  
**Формат:** теория (технология + best practices) → применение в BPMSoft (документация + исходники) → практика  
**Длительность:** ~5–6 месяцев при занятиях 3–4 раза в неделю

## Философия обучения

> **Ключевой принцип:** обучение строится на общих знаниях и лучших практиках по каждой технологии стека, с акцентом на их применение в BPMSoft.

### Роль документов
13 PDF-документов в проекте — это **контекст**, который помогает понять, как BPMSoft устроен изнутри: какие технологии используются, какие паттерны приняты, как организована архитектура. Документация описывает далеко не все аспекты платформы и **не является единственным источником знаний**.

### Подход к каждой теме
1. **Технология как таковая** — полноценное изучение с лучшими практиками, подводными камнями и реальными примерами из индустрии
2. **Применение в BPMSoft** — как эта технология используется в платформе, с опорой на документацию и исходники
3. **Практика** — задания, закрепляющие оба аспекта

### Формула
```
Сначала учим технологию (полноценно) → затем смотрим, как она работает в BPMSoft
Документы дополняют, но не ограничивают
```

## Блок 5а. Основы ExtJS в контексте BPMSoft (2–3 дня)

**Цель:** понимать систему классов ExtJS, которая лежит в основе всего клиентского кода BPMSoft.

### Зачем этот блок
BPMSoft использует фреймворк ExtJS как фундамент клиентской архитектуры. Без понимания `Ext.define` и `Ext.create` код модулей, схем и компонентов будет выглядеть как «магия». Этот блок даёт минимально необходимую базу — не весь ExtJS, а только то, что реально встречается в коде BPMSoft.

### Темы
- Что такое ExtJS и какую роль он играет в BPMSoft
- `Ext.define()` — объявление класса: `extend`, `alternateClassName`, `mixins`, `static`
- `Ext.create()` — создание экземпляра класса
- Наследование: `extend` и вызов `this.callParent(arguments)`
- Конфигурационные объекты и паттерн `{ bindTo: "..." }`
- Утилитные методы: `Ext.Array.each`, `Ext.isArray`, `Ext.isEmpty`

### Документация
- 📖 **Клиентская разработка**, стр. 26–33: «Класс модуля» — `Ext.define()`, свойства класса, инициализация экземпляров, наследование, переопределение членов
- 📖 **Клиентская разработка**, стр. 44–45: «Функция define()»

### Из исходников — примеры для разбора
```javascript
// 1. Объявление класса (Клиентская разработка, стр. 26)
Ext.define("BPMSoft.configuration.MyClass", {
    alternateClassName: "BPMSoft.MyClass",
    extend: "BPMSoft.BaseClass",
    myProperty: "значение",
    getMyProperty: function() {
        return this.myProperty;
    }
});

// 2. Создание экземпляра
var obj = Ext.create("BPMSoft.MyClass");
obj.getMyProperty(); // "значение"

// 3. Реальный пример из BPMSoft — создание запроса к данным
var select = Ext.create("BPMSoft.EntitySchemaQuery", {
    rootSchemaName: "Account"
});

// 4. Реальный пример — модуль с вызовом родительского метода
Ext.define("BPMSoft.configuration.BpmLoadingModule", {
    extend: "BPMSoft.BaseModule",
    init: function() {
        this.callParent(arguments); // вызов родительского init
    }
});
```

### Практика
- Прочитать стр. 26–33 и выписать все свойства, которые принимает `Ext.define()`
- Найти в исходниках CasePage 3 примера использования `Ext.create()` и объяснить, что создаётся
- Найти примеры `this.callParent(arguments)` и объяснить, зачем он нужен
- Составить шпаргалку: «`Ext.define` → объявить класс, `Ext.create` → создать экземпляр, `extend` → наследование, `callParent` → вызов родительского метода»
