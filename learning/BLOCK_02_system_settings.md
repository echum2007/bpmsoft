# Блок 2. Настройка системы (no-code)

**Цель:** научиться управлять внешним видом, структурой навигации и административными настройками BPMSoft без написания кода.

**Статус:** ✅ завершён

---

## 1. Дизайнер системы — точка входа

**Дизайнер системы** — центральный административный раздел BPMSoft. Всё административное управление системой доступно именно отсюда.

**Как попасть:** кнопка с шестерёнкой (⚙️) в правом верхнем углу интерфейса.

Дизайнер системы делится на блоки:

| Блок | Содержимое |
|------|-----------|
| Пользователи и права | Управление доступом, роли, операции |
| Настройка системы | Системные настройки, справочники |
| Настройка внешнего вида | Логотипы, рабочие места, цвета панели |
| Разработка | Конфигурация, пакеты, процессы |

---

## 2. Логотипы и корпоративная символика

BPMSoft позволяет заменить стандартные логотипы на брендированные.

**Путь:** Дизайнер системы → «Настройка корпоративной символики»

| Элемент | Где отображается | Рекомендуемый формат |
|---------|-----------------|----------------------|
| Логотип компании | Боковая панель (левый верх) | PNG, 300×100 px |
| Логотип на странице входа | Экран авторизации | PNG, 400×140 px |
| Логотип для главного меню | Страница входа, загрузка | PNG |
| Логотип на странице портала | Портал самообслуживания | PNG |
| Иконка (favicon) | Вкладка браузера | ICO/PNG, 16×16 или 32×32 px |

Соответствующие системные настройки (коды): `HeaderLogoImage`, `LogoImage`, `MenuLogoImage`, `SSPLogoImage`.

Практический смысл для Service Desk: при внедрении в организацию логотипы меняют в числе первых — это часть приёмки системы заказчиком.

## 3. Настройка цвета интерфейса

В BPMSoft есть несколько уровней управления цветом — от no-code до написания LESS-стилей.

### 3.1. Цвет боковой панели навигации (no-code)

**Путь:** Дизайнер системы ⚙️ → «Настройка внешнего вида» → «Настройка цвета панели разделов»

Требуется системная операция `CanManageSectionPanelColorSettings`.

Что настраивается:
- Цвет фона боковой панели разделов
- Цвет текста панели разделов
- Цвет выделенного (активного) раздела
- Цвет текста выделенного раздела

Выбор через полноценный color picker: поддерживаются HEX и RGB. После сохранения нужен **повторный вход** в систему для применения изменений. Кнопка «Восстановить настройки по умолчанию» возвращает исходные цвета BPMSoft.

### 3.2. Цветовое выделение строк и полей (no-code)

Механизм **«Правила цветового выделения»** — условное окрашивание записей в реестре или полей на странице по заданным условиям, без написания кода.

**Путь:** Рабочее место «Конструктор» → раздел «Правила цветового выделения»

Параметры правила:

| Поле | Описание |
|------|---------|
| Объект | Схема, к которой применяется правило |
| Область применения | Строки в разделе / строки в детали / поля на странице / все реестры |
| Условие | Как расширенный фильтр (аналогично фильтрации реестра) |
| Цвет фона | HEX-код или выбор из палитры |
| Цвет шрифта | HEX-код или выбор из палитры |
| Роль / Контакт пользователя | Для кого применяется правило |
| Приоритетность | Чем меньше число, тем выше приоритет при конфликте правил |
| Активно | Флаг включения/выключения правила |

**Важно:** правила цветового выделения не поддерживают колонки с типом «Дата» и «Дата и время» в условиях фильтрации. При использовании оператора `≠` в условии — допускается только одно значение.

**Примеры применения в Service Desk:**
- Обращения с критическим приоритетом → красный фон в реестре
- Обращения с нарушением SLA → оранжевый фон
- Закрытые обращения → серый шрифт
- Поле «Приоритет» на странице → цвет фона по значению

### 3.3. Тёмная тема

Полноценная альтернативная цветовая схема интерфейса (не просто инверсия).

На момент версии 1.8 находится в **бета-версии**: по умолчанию отключена, активируется через механизм **FeatureToggle** (управление функциональностью).

| Сценарий | Способ включения |
|----------|-----------------|
| Для одного пользователя | Страница управления функциональностью |
| Для группы пользователей | SQL-скрипт |

После вывода из бета-версии переключатель темы будет доступен всем пользователям в настройках профиля.

### 3.4. Кастомные стили через LESS (с кодом — Блок 4+)

Для изменения цветов за пределами боковой панели (кнопки, блоки, поля, фоны страниц) используются **LESS-модули** в конфигурации — это уже разработка, а не no-code.

Начиная с версии 1.8 в BPMSoft введены **CSS-переменные цветовых палитр**. При написании стилей рекомендуется использовать их, а не жёсткие HEX-значения — тогда стили корректно работают в обеих темах:

```css
.custom-status-button {
    background-color: var(--status-green-300);
    color: white !important;
    border-radius: 6px;
}

.custom-status-button:hover {
    background-color: var(--border-orange-bg-color);
}

/* Пример условной подсветки поля */
.custom-input-wrapper-warning {
    background-color: var(--status-red-300);
    border-radius: 8px;
}
```

Подробнее про LESS-модули — в **Блоке 4 (HTML/CSS)**.

## 4. Домашняя страница и дашборды

**Домашняя страница** — первое, что видит пользователь после входа. Состоит из настраиваемых виджетов.

### Виды виджетов

| Виджет | Назначение |
|--------|-----------|
| График | Линейный, столбчатый, круговой — по данным любого раздела |
| Список | Таблица с записями из раздела с фильтрами |
| Итоги | Одно агрегированное число |
| Воронка | Конверсия по стадиям |
| Виджет веб-страницы | Встроенный iframe с внешним URL |
| Виджет модуля | Кастомный Angular/ExtJS компонент |

### Уровни настройки

- **Для себя** — кнопка «Настроить страницу» прямо на домашней странице
- **Для роли/пользователей** — Дизайнер системы → «Настройка домашней страницы»

Типичный дашборд Service Desk: количество открытых обращений, SLA-нарушения, обращения по исполнителям, новые за сегодня.

## 5. Рабочие места

**Рабочее место** — набор разделов, сгруппированных в боковой панели навигации. По сути — «профиль интерфейса» для определённой роли.

**Путь:** Дизайнер системы → «Настройка рабочих мест»

### Структура рабочего места

| Элемент | Описание |
|---------|---------|
| Название | Отображается в выпадающем списке боковой панели |
| Иконка | SVG-иконка для идентификации |
| Разделы | Упорядоченный список разделов с иконками |
| Группы пользователей | Какие роли видят это рабочее место |

### Примеры рабочих мест в Service Desk

| Рабочее место | Разделы |
|--------------|---------|
| Оператор | Обращения, Контакты, Очередь обращений |
| Менеджер | Обращения, Итоги, Сервисные контракты, SLA |
| Администратор | Все разделы + Конструктор |

### Порядок разделов

Порядок разделов внутри рабочего места определяет порядок в боковой панели. Пользователь может скрыть ненужные разделы, но не может добавить те, которых нет в его рабочем месте. Для применения настроек — повторный вход в систему.

## 6. Системные настройки (SysSettings)

**Системные настройки** — именованные параметры конфигурации, влияющие на поведение системы. Активно используются в коде — это один из главных механизмов конфигурирования без жёстких значений в коде.

**Путь:** Дизайнер системы → «Системные настройки» (или рабочее место «Конструктор» → «Системные настройки»)

### Структура системной настройки

| Поле | Описание |
|------|---------|
| Название | Человекочитаемое имя |
| Код | Уникальный идентификатор (латиница, без пробелов) — используется в коде |
| Тип | Тип данных значения |
| Значение по умолчанию | Базовое значение |
| Описание | Комментарий |

### Типы значений

- Текст (краткий / длинный)
- Целое число / Дробное число
- Дата / Дата и время
- Логическое (да/нет)
- Справочник (ссылка на запись)
- Файл / Изображение

### Хранение в базе данных

| Таблица | Что хранит |
|---------|-----------|
| `SysSettings` | Код, название, тип — метаданные настройки |
| `SysSettingsValue` | Значения (в т.ч. разные для разных пользователей/ролей) |

При привязке системной настройки к пакету сначала переносится запись из `SysSettings`, затем отдельно запись из `SysSettingsValue`.

### Примеры системных настроек для Service Desk

```
Код: UsrSLAResponseTime          → время реакции по SLA в часах (Целое число)
Код: UsrDefaultServiceContract   → контракт по умолчанию (Справочник)
Код: UsrSendEmailOnCaseCreate    → отправлять email при создании (Логическое)
Код: UsrMaxCasePriority          → максимальный приоритет обращения (Целое число)
```

### Обращение к системным настройкам из кода

Это подробно рассматривается в Блоках 7–12, но важно знать паттерн заранее:

```javascript
// Клиентский JavaScript (Блок 7+)
BPMSoft.SysSettings.querySysSettingsItem("UsrSLAResponseTime", function(value) {
    console.log("SLA response time:", value);
}, this);
```

```csharp
// Серверный C# (Блок 12+)
var slaHours = BPMSoft.Core.Configuration.SysSettings.GetValue<int>(
    UserConnection, "UsrSLAResponseTime");
```

**Ключевое правило:** Код системной настройки — строка, по которой код обращается к ней. Выбирать коды нужно осмысленно, с префиксом организации (`Usr`, `CTI` и т.д.).

## 7. Справочники

**Справочники** (Lookups) — таблицы с управляемыми наборами значений, используемые как поля-ссылки в объектах.

**Путь:** Дизайнер системы → «Справочники»

### Примеры справочников в Service Desk

| Справочник | Примеры значений |
|-----------|-----------------|
| Статус обращения | Новое, В работе, Ожидание клиента, Закрыто |
| Приоритет обращения | Низкий, Средний, Высокий, Критический |
| Категория сервиса | Инцидент, Запрос на обслуживание, Изменение |
| Тип обращения | Телефон, Email, Портал, Чат |

Для каждого справочника можно добавлять/редактировать/удалять значения и настраивать дополнительные поля через мастер.

## Итоги блока

1. **Дизайнер системы** — единая точка входа для всего администрирования BPMSoft.

2. **Цвет интерфейса** — три уровня: боковая панель (no-code, сразу), условное выделение строк/полей (no-code, правила), глобальные стили (LESS-модули, Блок 4+). Тёмная тема — отдельный механизм через FeatureToggle.

3. **Рабочие места** — механизм персонализации интерфейса под роли. У оператора, менеджера и администратора Service Desk — разные наборы разделов.

4. **Системные настройки (SysSettings)** — ключевой механизм конфигурирования. Каждая настройка имеет уникальный **Код**, по которому к ней обращается код. `SysSettings` — метаданные, `SysSettingsValue` — значения.

5. **Справочники** — управляемые наборы значений для lookup-полей (статусы, приоритеты, категории обращений).

6. **Логотипы** — меняются через Дизайнер системы → «Корпоративная символика», не через код.
